# Truth 后续拓展规划

本文档用于说明阶段 6 完成后的下一步演进方向，便于持续迭代与验证效果。

---

## 1. 搜索质量优化

- **关键词提炼**：从页面正文中提取高频实体（人名、组织、地点、事件），提高搜索相关性  
- **多策略检索**：同时使用标题关键词、摘要关键词、实体关键词进行多路检索  
- **权重排序**：根据关键词命中次数、标题命中优先级对市场进行加权排序  
- **去噪过滤**：过滤过于宽泛或无意义的市场（如与正文不相关的热点）  

---

## 2. 数据展示增强

- **赔率趋势**：展示近 24h / 7d 概率变化趋势（若 API 支持）  
- **交易量变化**：显示日内成交量与总成交量对比  
- **流动性标签**：用“高/中/低”标签直观表示市场深度  
- **截止倒计时**：用倒计时方式显示市场截止时间  

---

## 3. 页面内体验优化

- **拖拽位置**：允许用户拖拽小组件位置并记忆  
- **自适应收起**：滚动页面时自动缩小为按钮，停留时展开  
- **轻量模式**：仅显示 top1~top3 相关市场，减少干扰  
- **页面冲突处理**：支持 Shadow DOM 或隔离命名空间  

---

## 4. 自定义偏好设置

- **市场类别过滤**：按政治 / 加密 / 体育 / 财经等分类筛选  
- **本地偏好**：在插件设置页保存用户偏好配置  
- **语言偏好**：支持中文页面优先、英文页面优先等策略  

---

## 5. 真实性标注与报告

- **置信度提示**：基于 Yes/No 概率给出简单的可信度指标  
- **结论卡片**：生成一句结论建议（例如：“市场倾向认为…可能性较高”）  
- **一键分享**：将市场截图或链接分享为报告卡片  

---

## 6. 技术结构演进

- **API 缓存**：对相同页面结果进行本地缓存，减少重复请求  
- **异步队列**：后台批量拉取市场数据与详情，避免阻塞  
- **错误回退**：API 不可用时降级展示缓存或提示  

---

## 7. 测试与验收建议

- 使用 `test/` 下的模拟页面，逐项验证展示与交互  
- 在真实新闻页与社交平台页上验证命中率  
- 记录“命中率 / 误匹配率 / 用户反馈”作为迭代依据  

---

## 8. 下一步建议实现顺序

1. 关键词提炼 + 搜索排序  
2. 页面内体验优化（拖拽、轻量模式）  
3. 数据展示增强（倒计时、趋势）  
4. 偏好设置与配置页  
5. 真实性标注与报告输出
